🔹 1. Why does print(line) work after the break?

It doesn't run after the break. That’s the key point.

Let’s step through the logic:

  • You get an input: line = input('> ')

  • If the user types "done", it breaks out of the loop before printing the line

  • If they type anything else, print(line) executes because the break didn't happen

Example:

shell
> hello
hello
> world
world
> done
Done!

→ print(line) only happens if the input was not "done".

🔹 2. Why does print('Done!') run in another indentation?

Because it's outside the while loop. This means it only runs once, after the loop ends (due to the break).

Think of it like this:

python
# Start loop
while True:
    # Ask for input
    line = input('> ')
    
    # If input is "done", break out of the loop entirely
    if line == 'done':
        break
    
    # Otherwise, print what the user typed
    print(line)

# This is outside the loop, runs once when loop ends
print('Done!')

🧠 Summary

  • print(line) is inside the loop and only runs if line != 'done'

  • break stops the loop before print(line) if the user types "done"

  • print('Done!') is after the loop, so it runs once after the loop ends
